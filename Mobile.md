# 墨韵 · 移动端适配规划

> 状态：**Phase A 已实施** · 最后更新：2026-06-11
>
> Phase A（布局可用 + 触控加固）已落地；Phase B（手感调参）待真机反馈后推进。

---

## 1. 目标与定位

### 1.1 适配目标

| 优先级 | 目标 |
|---|---|
| P0 | 触屏可作画，工具可选用，画廊可浏览 |
| P1 | 笔触手感针对手指 / 手写笔调优 |
| P2 | 布局美观，符合水墨气质 |
| **延后** | 性能优化、流体降频、分辨率降级 |

### 1.2 目标设备

- **主战场**：平板竖屏 / 横屏（iPad、Android 平板）
- **次战场**：手机竖屏（可用即可，不追求工笔级精细）
- **加分项**：Apple Pencil、主流 Android 手写笔（有则利用压感，无则降级）

### 1.3 非目标（保持克制）

- ❌ 不做原生 App / PWA（首版）
- ❌ 不做掌根拒绝、多指画布缩放
- ❌ 不做移动端专属笔刷系统
- ❌ 不做性能专项优化（见 §6，整节延后）
- ❌ 不为此引入 npm / 构建工具 / 文件拆分

## 2. UI 布局方案

桌面端保持现有 **左功能栏 + 两侧文房栏 + 中央画布** 不变。移动端实际落地为 **两侧抽屉 + 底部功能栏**，三者均可随时收展：

```
┌──────────────────────────────────────┐
│ ┌──┐                          ┌──┐  │
│ │笔│       画 布 区 域          │纸│  │ ← 左抽屉：笔·毫·墨·色
│ │墨│       （最大化）           │印│  │ ← 右抽屉：纸·水·款·印
│ └──┘                          └──┘  │
│        ┌──────────────────┐         │
│        │ 撤笔 涤纸 入藏 成图 画廊 │     │ ← 底部功能栏（可收起）
│        └──────────────────┘         │
└──────────────────────────────────────┘
```

- 三个把手（笔墨 / 纸印 / 功能）常驻屏幕边缘，点按收展；桌面端把手隐藏
- 默认：底部功能栏展开、两侧抽屉收起，画布可用宽度最大化
- 通过 `body` 上的 `m-left-open / m-right-open / m-actions-open` 类驱动，纯 CSS 过渡

### 2.1 底部文房栏（画笔 & 色彩类）

**收纳内容**（对应现有 `buildToolbar` 中的创作工具）：

| 分组 | 现有控件 | 移动端呈现 |
|---|---|---|
| 笔 | 4 种毛笔 | 横向排列，选中高亮，触区 ≥ 44px |
| 墨 | 8 级墨干湿 | 横向圆点条，可左右滑动 |
| 色 | 8 种颜色 | 横向圆点条，与墨同级切换或同排 |
| 水 | 清水笔 | 独立按钮，选中态与桌面一致 |
| 纸 | 4 种宣纸 | 小色块横排，或收入二级面板 |

**交互**：

- 默认展开一行（当前选中项可见）
- 点击分组标签或把手 → 向上展开完整面板
- 展开时画布区域等比缩小，不遮挡画布中心
- 面板带半透明背景，水墨风格延续桌面 CSS 变量

**实现提示**：

- 纯 CSS：`@media (max-width: 768px)` 或 `(pointer: coarse)` 切换布局
- `#toolbar` 拆为两个容器：`#toolbar-create`（底部）、`#toolbar-actions`（左侧），桌面端 CSS 合并回单栏；JS 侧 `buildToolbar()` 分组构建，避免重复逻辑

### 2.2 左侧功能栏（操作类）

**收纳内容**：

| 按钮 | 现有对应 | 说明 |
|---|---|---|
| 题款 | `openInscribe` | 保留模态框 |
| 撤笔 | `undo` | 可加快捷条提示 |
| 涤纸 | `clearPaper` | 需二次确认（触屏易误触） |
| 入藏 | `saveCurrent` | 主操作，视觉强调 |
| 成图 | `exportPNG` | |
| 画廊 | `backToGallery` | |

**交互**：

- 默认 **收起**：仅露窄条（≈ 36px）或悬浮圆钮
- 点击展开浮层 / 抽屉，不永久占宽
- 收起态下图布可用宽度接近全屏

### 2.3 画廊视图（移动端）

| 桌面 | 移动 |
|---|---|
| 左网格 + 右竖排品牌 | 顶栏横排品牌 + 下方单列卡片 |
| 172px 网格 | 2 列或单列（视屏宽） |
| hover 显示删除 | 长按或卡片内常驻删除钮 |
| 大留白 padding | `padding: 24px 16px` |

### 2.4 绘制页其他元素

| 元素 | 处理 |
|---|---|
| 竖排题名 `#work-title` | 缩小字号；或收入顶部折叠条 |
| 笔锋光标 `#cursor-ring` | **触屏隐藏**（`pointer: coarse` 或 `pointerType === 'touch'`） |
| 题款模态框 | 全宽底部弹出 Sheet 样式，避免键盘遮挡 |
| Toast | 位置略上移，避开底部文房栏 |

### 2.5 断点建议

```css
/* 平板 */
@media (max-width: 1024px) { ... }

/* 手机 */
@media (max-width: 768px) { ... }
```

优先用宽度断点；`(pointer: coarse)` 作辅助，避免触控笔记本误伤。

## 3. 手感适配（P1 核心）

性能暂缓，**手感是移动端除 UI 外最值得投入的部分**。

### 3.1 当前桌面模型（需知）

```
速度 vel ← 采样点间距 / 时间差
粗细 ← brush.size × wet.spread × f(vel)
干燥 ← wet.dry + vel × 系数
平滑 ← lerp(last, current, 0.55~0.7)
```

该模型为鼠标高频采样调参，触屏直接沿用会 **偏抖、偏细、偏枯**。

### 3.2 触屏检测

```javascript
// 建议封装
const isTouch = () => matchMedia('(pointer: coarse)').matches;
// 运行时
e.pointerType === 'touch' | 'pen'
```

`pen` 与 `touch` 分支处理：手写笔可走压感，手指走速度曲线。

### 3.3 手指模式调参方向

| 参数 | 桌面 | 移动（手指）建议 |
|---|---|---|
| 平滑 lerp 系数 | 0.55～0.7 | **0.35～0.5**（更强平滑） |
| 速度衰减 | `vel * 0.72 + …` | 加大惯性，过滤尖峰 |
| 最小笔画宽度 | `max(1.2, …)` | **提高下限**（蘸指较粗） |
| 工笔 `fine` | 细线优先 | 手机端默认笔刷改为 **写意** |
| 起笔宽度 | `curW * 0.35` | 略提高，减少「点一下没痕迹」 |
| 采样距离阈值 | `d < 1` return | 触屏可降至 `d < 0.5` |

以上数值需在真机上调，文档仅定方向。

### 3.4 手写笔模式（有则启用）

利用 Pointer Events 扩展字段：

| 字段 | 用途 |
|---|---|
| `pressure` | 0～1 映射透明度 / 宽度（减轻对速度的依赖） |
| `pointerType === 'pen'` | 启用压感，恢复较高工笔精度 |
| `tiltX / tiltY` | **首版不做**，后续可选 |

压感映射示意：

```
有效宽度 = base × (0.4 + pressure × 0.6)
有效干燥 = wet.dry + (1 - pressure) × 0.2
```

### 3.5 触控事件加固

| 项 | 做法 |
|---|---|
| 丢点 | `pointerdown` → `setPointerCapture(e.pointerId)` ✅ |
| 页面滚动 | 画布 `#paper` 设 `touch-action: none` ✅ |
| 误触缩放 | iOS 10+ 忽略 `user-scalable=no`，**不依赖它**；改用 `touch-action: manipulation`（禁双击缩放）+ 画布 `touch-action: none` ✅ |
| 双指 | 首版忽略；不阻止默认，避免与系统手势硬刚 |
| 抬笔 | `pointerup` + `pointercancel` 均触发 `commitStroke` ✅ |

### 3.6 水洗 / 题款在触屏上

- **水洗**：手指大面积涂抹反而自然，保持现有逻辑，仅加大 `washRadius` 触屏系数（可选）
- **题款**：点选落点用十字准星替代鼠标；印章尺寸随 DPI 略放大

## 4. 实施阶段（建议顺序）

与桌面优化并行不冲突；**移动端建议等桌面稳定后再开工**。

### Phase A — 布局可用 ✅ 已完成

- [x] 媒体查询（≤920px）：画廊窄屏网格 / 绘制页画布满幅
- [x] 两侧文房抽屉 + 底部功能栏，三把手随时收展
- [x] 触区放大（按钮 ≥44px、圆点 27px）
- [x] 触屏隐藏 `#cursor-ring`（`pointerType` 运行时判断 + 媒体查询双保险）
- [x] `touch-action: none` + `setPointerCapture` + `pointercancel` 提交笔画
- [x] 触屏撤销快照上限降为 5（防 iOS Safari 内存闪退，提前自 §6）
- [x] 墨/色圆点触屏选中后 Toast 显示名称（hover 提示在触屏不可用）
- [x] 流体静止时跳过整格扫描（顺手做，桌面端同样受益）
- [x] 题款/钤印弹窗改为底部 Sheet；输入框 ≥16px 防 iOS 聚焦放大
- [x] Toast 上移避开底部功能栏

**验收**：手机浏览器能选笔、选色、画几笔、存画廊。（待真机验证）

### Phase B — 手感调优（预估 3～5 天）

- [ ] `pointerType` 分支与触屏平滑参数
- [ ] 手指 / 手写笔双模式（压感可选）
- [ ] 默认笔刷、墨级推荐（移动偏写意 + 浓墨）
- [ ] 涤纸等危险操作加确认
- [ ] 题款 Sheet 化、键盘友好

**验收**：平板 + Pencil 写意流畅；手机手指涂鸦无严重断线。

### Phase C — 打磨（可选）

- [ ] 横竖屏切换布局微调
- [ ] 展开/收起动画
- [ ] 首次打开触屏引导（「建议使用手写笔」）

### Phase D — 性能（整节延后）

见 §6，不在首版范围内。

## 6. 性能优化（延后备忘）

以下问题已知，**首版不处理**；若真机严重卡顿再逐项开启：

| 项 | 现状 | 未来手段 |
|---|---|---|
| 流体双步迭代 | 每帧 2×87000 格 | 移动端改为 1 步；静止时跳过 |
| 画布分辨率 | 固定 1000×1390 | 内部 0.5x 渲染 |
| 撤销快照 | 全尺寸 ImageData ×6 | 降分辨率快照 |
| 水洗 blur | `getImageData` + filter | 降低半径 / 降频 |
| localStorage | 全 PNG | 已有 0.55 倍降级，可再加 |

## 7. 风险与对策

| 风险 | 对策 |
|---|---|
| 手机工笔不可用 | 产品预期内；引导写意 / 泼墨 |
| 中低端机卡顿 | 延后性能；首版接受 |
| 触屏笔记本布局突变 | 断点以宽度为主，`pointer: coarse` 为辅 |
| 维护两套布局 CSS | 变量复用，仅布局规则分叉 |
| 与桌面优化冲突 | 先桌面后移动；CSS 媒体查询隔离 |
